## 2. The one actual diff — `src/hdgl_bootloaderz.h`

```diff
--- conscious-128-bit-floor/metal_infer_for_primes/hdgl_bootloaderz.h
+++ v0.3/src/hdgl_bootloaderz.h
@@ -12,6 +12,13 @@
 #include <stdlib.h>
 #include <math.h>
 
+/* MSVC does not support __attribute__; neutralise it for portability */
+#ifdef _MSC_VER
+#  ifndef __attribute__
+#    define __attribute__(x)
+#  endif
+#endif
+
 #define HDGL_VERSION_MAJOR 2
```

**What it does:** Adds an MSVC portability shim so `__attribute__(...)` is silently discarded on Windows/MSVC builds. This is consistent with the new `Makefile.win` and `src/tb_win32.h` also appearing in v0.3.

**Action:** Keep the v0.3 version. This is a pure additive portability fix — safe to carry forward.

## 5. Consolidation action plan

1. **No merges needed.** The only changed shared file (`hdgl_bootloaderz.h`) is a clean additive patch in v0.3 — keep it.

2. **v0.3 is the canonical source.** The baseline `metal_infer_for_primes/` subtree is superseded. If you're maintaining a mono-repo, the move is:
   - Archive/tag the `conscious-128-bit-floor` snapshot as `v0.2-archive`
   - Promote `v0.3/` as the new project root

3. **Dropped files to consider:** If `image_caption.c` or `hdgl_corpus_seeder.c` still need a home, they'd need to be ported to the new layer architecture (likely under `layer3/` as orchestration tools). They're not wired in currently.

4. **Third-party deps** (`miniz`, `linenoise`, `jsmn`) — if needed downstream, add as git submodules or copy to a `third_party/` dir rather than in-tree.
